Copy as Markdown In this article Enabling the Parent issue field Enabling the Sub-issue progress field If your organization uses sub-issues, you can enable the "Parent issue" and "Sub-issue progress" fields on your projects to see the relationships between your issues and the progress made on those issues. The "Parent issue" field can be used to group items, allowing you to create views that break down your work using the sub-issue hierarchies you have created. For more about grouping views, see Changing the layout of a view . You can also filter by the "Parent issue" field to only display items that are sub-issues of a particular issue. You can start by typing "parent-issue" and then selecting an issue from the list. Or, if you know the repository and issue number, you can type the filter in full: parent-issue:"&#x3C;OWNER>/&#x3C;REPO>#&#x3C;ISSUE NUMBER>" To use the filter, replace &#x3C;OWNER> with the repository owner, &#x3C;REPO> with the repository name, and &#x3C;ISSUE NUMBER> with the issue number. See Filtering projects . Enabling the Parent issue field You can enable the "Parent issue" field to see which parent issues the issues in your project belong to.
